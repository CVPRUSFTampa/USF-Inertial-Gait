Downloading and Building the ROC Generator

  1. Download the source code from SourceForge. This is done by checking out the code using SVN. SVN is available on CVRL and the other compute nodes. To check out the code, enter the following command :

    svn co https://cvrlroc.svn.sourceforge.net/svnroot/cvrlroc cvrlroc

    This will create a directory called cvrlroc containing the code.

  2. Navigate to the c++ folder in the newly created cvrlroc directory. This directory contains all of the code needed by the ROC Generator. No other libraries are needed.

  3. Execute

    make

    This will compile the code into executables found in the bin directory.

  4. Navigate to the bin directory.


How to Generate a ROC Curve

The ROC generator is used as follows :
./generateROC [scoreFile] [rocFile] [label] [numBoot] [numRocPoints]
Where the arguments are

  1. [scoreFile] is a file containing the match and nonmatch scores. Each line of score file should contain information about a single score in the form of [match],[score]where [match] is a 0 indicating nonmatch or a 1 indicating match, and [score] is simply the score. An example line would be
    1,0.124165
  2. [rocFile] is the output file that contains the R vectors necessary to plot the ROC curve. The following vectors are provided :
    1. meanTA : the average TAR for each threshold used to compute the ROC curve.
    2. meanFA : the average FAR for each threshold used to compute the ROC curve.
    3. stdTA : the standard deviation of the TARs of the bootstraps at each point. If enough bootstraps are used, plotting error bars of two standard deviations closely approximates a 95% confidence interval.
    4. stdFA : the standard deviation of the FARs.
    5. CITA_low : The TAR at the 5% level of the bootstrap distribution for each ROC point.
    6. CITA_high : The TAR at the 95% level of the bootstrap distribution for each ROC point.
    7. CIFA_low : The FAR at the 5% level of the bootstrap distribution for each ROC point.
    8. CIFA_high : The FAR at the 95% level of the bootstrap distribution for each ROC point.
  3. [label] : a label appended to each of the above R vectors. This allows multiple ROC R files to be source by a single R script for plotting.
  4. [numBoot] : the number of bootstraps to perform.
  5. [numRocPoints] : the number of points desired on the each ROC curve.
